# CMS Migration 0025 Diff

Migration file: `cms-migrations.ts` — `migration_0025_session_events_type_filter`

Adds server-side event-type filtering for session-event history paging. Chat
history is sparse in the raw event stream (busy sessions have thousands of
tool/orchestration events between chat messages), so clients paging backward
for chat previously had to drain raw pages. Clients now pass the renderable
message types and get transcript-dense pages.

## Table Changes

None.

## New Indexes

```sql
CREATE INDEX IF NOT EXISTS idx_SCHEMA_events_session_type_seq
    ON SCHEMA.session_events (session_id, event_type, seq);
```

Composite btree serving the filtered scans. A partial index cannot be used
here: the filter arrives as a parameterized `event_type = ANY($4)`, which the
planner cannot prove implies a partial-index predicate.

## New Stored Procedures

Both are 4-arg OVERLOADS of the bounded 3-arg versions from migration 0013.
The 3-arg versions are left in place so older workers/portals keep working
mid-rollout (`CREATE OR REPLACE FUNCTION` cannot change a signature anyway).

- `cms_get_session_events(p_session_id TEXT, p_after_seq BIGINT, p_limit INT, p_event_types TEXT[])`
- `cms_get_session_events_before(p_session_id TEXT, p_before_seq BIGINT, p_limit INT, p_event_types TEXT[])`

Bodies are identical to the 0013 bounded versions plus one predicate in each
WHERE clause:

```diff
 CREATE OR REPLACE FUNCTION SCHEMA.cms_get_session_events(
     p_session_id  TEXT,
     p_after_seq   BIGINT,
-    p_limit       INT
+    p_limit       INT,
+    p_event_types TEXT[]
 ) RETURNS SETOF SCHEMA.session_events AS $$
 DECLARE
     v_limit INT := GREATEST(1, LEAST(COALESCE(p_limit, 1000), 1000));
 BEGIN
     IF p_after_seq IS NOT NULL AND p_after_seq > 0 THEN
         RETURN QUERY
         SELECT * FROM SCHEMA.session_events
         WHERE session_id = p_session_id AND seq > p_after_seq
+          AND (p_event_types IS NULL OR event_type = ANY(p_event_types))
         ORDER BY seq ASC LIMIT v_limit;
     ELSE
         RETURN QUERY
         SELECT * FROM (
             SELECT * FROM SCHEMA.session_events
             WHERE session_id = p_session_id
+              AND (p_event_types IS NULL OR event_type = ANY(p_event_types))
             ORDER BY seq DESC LIMIT v_limit
         ) t ORDER BY seq ASC;
     END IF;
 END;
 $$ LANGUAGE plpgsql;
```

```diff
 CREATE OR REPLACE FUNCTION SCHEMA.cms_get_session_events_before(
     p_session_id  TEXT,
     p_before_seq  BIGINT,
-    p_limit       INT
+    p_limit       INT,
+    p_event_types TEXT[]
 ) RETURNS SETOF SCHEMA.session_events AS $$
 DECLARE
     v_limit INT := GREATEST(1, LEAST(COALESCE(p_limit, 1000), 1000));
 BEGIN
     RETURN QUERY
     SELECT * FROM (
         SELECT * FROM SCHEMA.session_events
         WHERE session_id = p_session_id AND seq < p_before_seq
+          AND (p_event_types IS NULL OR event_type = ANY(p_event_types))
         ORDER BY seq DESC LIMIT v_limit
     ) t ORDER BY seq ASC;
 END;
 $$ LANGUAGE plpgsql;
```

## Changed Stored Procedures

None (the 3-arg versions are untouched).

## TypeScript Callers

`PgSessionCatalog.getSessionEvents` / `getSessionEventsBefore` accept an
optional `eventTypes: string[]`; when present they call the 4-arg overload and
fall back to the 3-arg proc on Postgres error 42883 (DB predates 0025 — the
portal does not run migrations, so it must tolerate an older database).
