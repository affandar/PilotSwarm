# CMS Migration 0023 Diff

Reapplies the corrected turn-metrics stored-procedure layer from migration 0022 under a new migration version so already-migrated schemas receive the fixes without a reset.

Changes delivered by the corrected procedure layer:

- `cms_update_session_group` now accepts an `owner` patch, allowing management to set an empty group's adopted owner before moving the first owned session into it.
- `cms_get_session_tree_stats_by_model` falls back to `session_metrics` token totals for sessions that do not yet have `session_turn_metrics` rows.
- `cms_get_fleet_stats_by_agent` falls back to summary token totals for summary-only sessions while continuing to use turn metrics when present.
- `cms_get_user_stats_by_model` applies the same summary-only fallback for owner/user aggregates.

This keeps real turn metrics authoritative when they exist, while preserving existing direct CMS tests and legacy summary-only rows.
