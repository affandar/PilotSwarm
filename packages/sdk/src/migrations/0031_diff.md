## Function Changes

### `cms_list_users` — new

```diff
+CREATE OR REPLACE FUNCTION SCHEMA.cms_list_users(
+    p_limit INT
+) RETURNS TABLE (
+    provider     TEXT,
+    subject      TEXT,
+    email        TEXT,
+    display_name TEXT
+)
```

Behavior:

- clamps `p_limit` to `1..2000`, defaulting to `500`;
- excludes the synthetic `system/system` and `local/default` principals;
- returns only users with a non-null display name, which distinguishes a real
  sighting from a placeholder row created solely by a targeted grant;
- orders by newest `updated_at`, then newest `user_id`;
- exposes provider, stable subject, email, and display name for owner/admin
  share-management autocomplete.

## TypeScript Callers

- `PgSessionCatalogProvider.listKnownUsers()` calls this procedure.
- `PilotSwarmManagementClient.listKnownUsers()` and the Web API/MCP sharing
  surfaces expose the bounded result to authenticated users for grantee lookup.

The directory is a lookup helper, not an allowlist. Migration `0032` handles
adoption of email-keyed placeholder grants when the real principal first signs
in; migration `0033` ensures the grant path cannot overwrite a sighted user's
profile fields.
