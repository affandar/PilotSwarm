# CMS Migration 0026 Diff

Migration file: `cms-migrations.ts` — `migration_0026_session_splash_mobile`

Adds a narrow-viewport splash variant. Agents ship an optional `splashMobile`
banner; the UI swaps it in when the main splash art is wider than the pane
(mobile portal, narrow terminals).

## Table Changes

- `sessions`: adds nullable `splash_mobile TEXT`.

## New Indexes

None.

## New Stored Procedures

- `cms_create_session(p_session_id TEXT, p_model TEXT, p_reasoning_effort TEXT, p_parent_session_id TEXT, p_is_system BOOLEAN, p_agent_id TEXT, p_splash TEXT, p_group_id TEXT, p_splash_mobile TEXT)` —
  9-arg OVERLOAD of the 8-arg group-aware version (baseline 0016 range);
  identical body plus `splash_mobile` in the insert column list and the
  ON CONFLICT revival SET. The 8-arg version stays so old workers keep
  working mid-rollout.

## Changed Stored Procedures

- `cms_update_session` (vs 0024): one new jsonb column rule —

  ```diff
           splash            = CASE WHEN p_updates ? 'splash'          THEN (p_updates->>'splash')       ELSE splash            END,
  +        splash_mobile     = CASE WHEN p_updates ? 'splashMobile'    THEN (p_updates->>'splashMobile') ELSE splash_mobile     END,
           active_turn_index = CASE WHEN (p_updates ? 'state') AND (p_updates->>'state') <> 'running' THEN NULL ELSE active_turn_index END,
  ```

  Needed because the spawn paths (`spawnChildSession`, `spawn_agent`) set
  splash via the jsonb meta update, not at create time. No signature change.

- `cms_get_session` (vs 0024): DROP + CREATE with `splash_mobile TEXT`
  appended to the `RETURNS TABLE` column set and `sess.splash_mobile` to the
  select list. Required because `RETURNS TABLE` is a fixed list.

- `cms_list_sessions` (vs the owner-join version): DROP + CREATE with
  `splash_mobile TEXT` appended, same reason.

- `cms_list_group_sessions` (vs the owner-join version): DROP + CREATE with
  `splash_mobile TEXT` appended. Must move in lockstep with
  `cms_list_sessions` — its body is `SELECT * FROM cms_list_sessions()` into
  its own fixed column list, so differing shapes fail at runtime with
  "structure of query does not match function result type".

- `cms_list_sessions_page` is intentionally NOT touched: it returns
  `SETOF sessions`, so the new table column flows through automatically.

## TypeScript Callers

`PgSessionCatalog.createSession` passes `splashMobile` via the 9-arg overload
(falling back to the 8-arg proc on Postgres 42883 when the DB predates 0026);
`updateSession` forwards `updates.splashMobile` through the jsonb payload;
`rowToSessionRow` maps `row.splash_mobile`.
