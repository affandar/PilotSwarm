## Table Changes

### `session_group_owners` - new

Matches the existing `session_owners` normalized ownership shape for session groups.

```sql
CREATE TABLE IF NOT EXISTS SCHEMA.session_group_owners (
    group_id    TEXT PRIMARY KEY REFERENCES SCHEMA.session_groups(group_id) ON DELETE CASCADE,
    user_id     BIGINT NOT NULL REFERENCES SCHEMA.users(user_id),
    assigned_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

Existing `session_groups.owner` JSONB values are backfilled into `users` and `session_group_owners` so older group rows keep their owner identity after the normalized schema is introduced.

## New Indexes

### `cms_assign_session_group` - new

```diff
+ cms_assign_session_group(
+     p_session_id TEXT,
+     p_group_id TEXT DEFAULT NULL
+ ) RETURNS VOID
```

Validates that a non-system session and target group have matching normalized owner keys before updating `sessions.group_id`. Unowned sessions can only move into unowned groups. Passing `NULL` removes the session from its group.
