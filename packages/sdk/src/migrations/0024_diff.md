# CMS Migration 0024 Diff

Migration file: `cms-migrations.ts` — `migration_0024_stop_turn_active_turn_index`

Adds in-flight turn tracking for the stop-turn feature (`docs/proposals-impl/stop-button-turn-abort-plan.md`).

## Table Changes

- `sessions`: adds nullable `active_turn_index INTEGER`. Set while a turn is in flight; null otherwise. Used by `stopSessionTurn()` to address the turn-scoped stop queue `stopTurn.<turnIndex>`.

## New Stored Procedures

- `cms_set_active_turn_index(p_session_id TEXT, p_turn_index INTEGER)` — pre-turn writeback from the `runTurn` activity: publishes the in-flight turn index and bumps `updated_at`.

## Changed Stored Procedures

- `cms_get_session` (vs the 0016 owner-join version): DROP + CREATE with `active_turn_index INTEGER` appended to the `RETURNS TABLE` column set and `sess.active_turn_index` to the select list. Required because `RETURNS TABLE` is a fixed column list — the new table column is not returned automatically. `cms_list_sessions` / `cms_list_sessions_page` are intentionally NOT extended: list surfaces key Stop off `state === 'running'`; only the single-session read needs the turn index.

- `cms_update_session` (vs 0022/0023): adds one column rule —
  `active_turn_index = CASE WHEN (p_updates ? 'state') AND (p_updates->>'state') <> 'running' THEN NULL ELSE active_turn_index END`.
  Any state transition away from `running` (orchestration stop bookkeeping, error transitions, idle) retires the stop-queue target automatically. No signature change; all other column rules unchanged.

- `cms_complete_turn_writeback` (vs 0022/0023): the sessions UPDATE additionally sets `active_turn_index = NULL` — the turn has ended, whatever the result type. No signature change; metrics upsert, turn-metrics insert, and `session.turn_completed` event emission unchanged.
