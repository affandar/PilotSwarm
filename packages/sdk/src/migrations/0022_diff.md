# 0022 ‚Äî turn_metrics_reasoning_effort

## Table

- `session_metric_summaries` is physically renamed to `session_metrics` when the old
  table exists and the target does not. A compatibility view named
  `session_metric_summaries` remains for older read/update procedure bodies.
- `session_turn_metrics`: add nullable `reasoning_effort TEXT`.

## Backfill

- Insert one synthetic `legacy_summary` row into `session_turn_metrics` only for
  sessions that have nonzero summary tokens and no existing turn rows. This avoids
  double-counting deployments that already have real per-turn rows.

## Indexes (new)

- `session_metrics(agent_id, model)`
- `session_metrics(agent_id, model, reasoning_effort)`
- `session_metrics(parent_session_id)`
- `session_metrics(updated_at DESC)`
- `(session_id, model, reasoning_effort)`
- `(model, reasoning_effort, started_at DESC)`
- `(agent_id, model, reasoning_effort, started_at DESC)`

Old `idx_*_sms_*` indexes are dropped after replacement indexes are created.

## Stored procedures

- `cms_get_session_metric_summary` ‚Äî now returns from `session_metrics`.
- `cms_create_session`, `cms_update_session`, `cms_upsert_session_metric_summary`,
  `cms_prune_deleted_summaries` ‚Äî recreated against `session_metrics`.
- `cms_insert_turn_metric` ‚Äî gains `p_reasoning_effort` (now 17 args, inserted after model,
  trimmed-to-null). Callers must pass the new arg.
- `cms_complete_turn_writeback` (new) ‚Äî atomically updates `sessions`, increments
  `session_metrics`, inserts `session_turn_metrics`, and records an enriched
  `session.turn_completed` event.
- `cms_get_session_turn_metrics` ‚Äî output adds `reasoning_effort`.
- `cms_get_session_tokens_by_model` (new) ‚Äî per-session token totals + turn count grouped
  by the `model:effort` label (`provider:model:reasoning_effort`, `(unknown)` fallbacks).
- `cms_get_session_tree_stats_by_model`, `cms_get_fleet_stats_by_agent`,
  `cms_get_user_stats_by_model` ‚Äî token totals and `turn_count` now come from
  `session_turn_metrics`; session/non-token counters remain on `session_metrics`.

## TypeScript callers

- `InsertTurnMetricInput` / `TurnMetricRow`: add `reasoningEffort`.
- `CompleteTurnWritebackInput` + `completeTurnWriteback()` added to `SessionCatalog`.
- Tree/fleet/user model bucket shapes include per-bucket `turnCount`.
