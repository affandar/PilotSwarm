# CMS Migration 0016 Diff: Base Infrastructure State Compatibility Fixes

## Scope

Applies stored-procedure fixes introduced after `0015` so schemas that already recorded migration `0015` still receive the corrected behavior.

## Function Changes

### `cms_create_session` — compatibility shim

Adds/replaces the 7-argument `cms_create_session(...)` overload. It delegates to the 8-argument group-aware implementation with `p_group_id = NULL`, preserving existing callers while allowing parent-child group inheritance to run inside the new implementation.

### `cms_update_session_summary` — normalized short summaries

Replaces the procedure so explicit and derived short summaries collapse whitespace, trim empty values, and cap stored `short_summary` at 240 characters.

### `cms_list_sessions` — stable CMS ordering

Drops and recreates the widened read procedure with `updated_at DESC, session_id DESC` ordering. Summary freshness remains available as data, but CMS read ordering stays aligned with bounded page cursors.

### `cms_create_session_group` — duplicate detection

Replaces the procedure without `ON CONFLICT DO NOTHING`, so duplicate group creation raises the underlying unique-key error instead of pretending a new group was created.

### `cms_list_group_sessions` — stable group-member ordering

Replaces group-member reads to order by `updated_at DESC, session_id DESC`, matching full and paged session reads instead of using stale summary freshness.
