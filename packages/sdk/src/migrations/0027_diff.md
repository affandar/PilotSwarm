# CMS Migration 0027 Diff

Migration file: `cms-migrations.ts` — `migration_0027_session_raw_size_bytes`

Records the uncompressed snapshot size alongside the compressed one. Under
the brotli-4 codec switch, stored snapshot bytes drop ~10–15× for
JSONL-heavy sessions; `raw_size_bytes` keeps capacity trends continuous
across the codec change and lets the Stats pane show stored / raw / ratio.

## Table Changes

- `session_metrics`: adds `raw_size_bytes BIGINT NOT NULL DEFAULT 0`.

## View Changes

- `session_metric_summaries` (the backward-compat view created at 0022 when
  `session_metric_summaries` was renamed to `session_metrics`): DROP +
  CREATE so its frozen `SELECT *` column list picks up `raw_size_bytes`.
  Postgres views do not auto-track new base-table columns. Guarded by an
  `information_schema.views` existence check so schemas where the object is
  still a table (pre-rename) are untouched.

## New Indexes

None.

## Changed Stored Procedures

- `cms_upsert_session_metric_summary` (CREATE OR REPLACE, no signature
  change): reads `rawSizeBytes` from the jsonb payload and sets it
  absolutely (mirrors `snapshotSizeBytes` — `WHEN p_updates ? 'rawSizeBytes'
  THEN v_raw ELSE …`). The INSERT column list and ON CONFLICT SET gain
  `raw_size_bytes`.

- `cms_get_session_tree_stats(TEXT)` (DROP + CREATE): `RETURNS TABLE` gains
  `total_raw_size_bytes BIGINT`; the SELECT adds
  `COALESCE(SUM(m.raw_size_bytes), 0)`. Feeds the Stats pane TREE block.
  DROP required — `RETURNS TABLE` is a fixed list.

- `cms_get_fleet_stats_totals(BOOLEAN, TIMESTAMPTZ)` (DROP + CREATE): same
  addition; feeds the Stats pane Fleet-tab totals.

## Deferred (fast-follow)

The per-model / per-agent breakdown rollups —
`cms_get_session_tree_stats_by_model`, `cms_get_fleet_stats_by_agent`,
`cms_get_user_stats_by_model` — are NOT changed here. They still return
`total_snapshot_size_bytes` only; the TypeScript mappers read
`total_raw_size_bytes` with a `|| 0` fallback, so those breakdown cards show
raw = 0 until a follow-up migration adds the SUM. The primary per-session
PERSISTENCE block, the session-tree rollup, and the fleet totals (the three
surfaces on the screenshotted Stats pane) are fully wired.

## TypeScript Callers

- `SessionMetricSummary` gains `rawSizeBytes`; `SessionMetricSummaryUpsert`
  gains `rawSizeBytes?`; `rowToSessionMetricSummary` maps `raw_size_bytes`.
- `SessionTreeStats.tree` gains `totalRawSizeBytes`; `FleetStats.totals`
  gains `totalRawSizeBytes`; both mappers read `total_raw_size_bytes`.
- The runTurn commit summary write (`session-proxy.ts`) sends `rawSizeBytes`
  from the store commit result; the store commit/hydrate results carry it
  from the compression pipeline (uncompressed tar-stream length).
- UI (`selectors.js`): PERSISTENCE + TREE cards show `Uncompressed` and a
  `Compression Nx` line derived from raw / stored.
