# Diff for CMS migration 0010

Migration file: `cms-migrations.ts` — `migration_0010_user_profile_and_copilot_key`

## Table Changes

### `users` — modified

```diff
  user_id              BIGSERIAL PRIMARY KEY,
  provider             TEXT NOT NULL,
  subject              TEXT NOT NULL,
  email                TEXT,
  display_name         TEXT,
+ profile_settings     JSONB NOT NULL DEFAULT '{}'::jsonb,
+ github_copilot_key   TEXT,
  created_at           TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at           TIMESTAMPTZ NOT NULL DEFAULT now()
```

`profile_settings` is an opaque JSON document the application owns — it
holds the per-user UI preferences that previously lived only in client
storage (theme id, pinned session ids, layout adjustments, owner
filter, etc.). Setters replace the document wholesale.

`github_copilot_key` is the optional per-user override for the
`github-copilot` model provider token. When set, the worker prefers it
over the env-supplied `GITHUB_TOKEN` for sessions owned by this user.
A `NULL` value clears the override.

The key is stored as plaintext today, matching how the worker reads
`GITHUB_TOKEN` from `.env`. Encryption-at-rest is a follow-up.

## New Stored Procedures

### `cms_get_user_profile(provider, subject)` — new

Public read used by the management API and Admin Console. Returns the
user row, the JSONB `profile_settings` document, and a boolean
`github_copilot_key_set` flag. The actual key text is **not** returned
through this procedure; the management surface should never expose it.

Returns zero rows when the user is unknown so callers can render an
"unconfigured" admin state without first registering the user.

### `cms_get_user_github_copilot_key(provider, subject)` — new

Internal-only read for the worker's per-user token resolver. Returns
the raw key text (or `NULL`). Never expose this through any public
management API surface.

### `cms_set_user_profile_settings(provider, subject, email, display_name, settings)` — new

Creates the user row via `cms_register_user` if it does not yet exist
(so preferences can be saved before the user has created any sessions),
then **replaces** `profile_settings` with the supplied JSONB document.
A `NULL` document is normalized to `'{}'`.

Refreshes `updated_at`.

### `cms_set_user_github_copilot_key(provider, subject, email, display_name, key)` — new

Creates the user row via `cms_register_user` if missing, then sets or
clears `github_copilot_key`. A `NULL` or all-whitespace key clears the
override and reverts the user to the worker's env-supplied default
token.

Refreshes `updated_at`.

None to existing functions.
